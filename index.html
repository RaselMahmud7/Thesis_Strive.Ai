Executive Summary

This thesis explores how Large Language Models (LLMs) can generate dynamic UI modules for personalized learning dashboards, addressing the static nature of current educational platforms.

In collaboration with Digital Ops and Strive AI, the study compares traditionally designed modules with LLM-generated ones through user studies using quantitative metrics and qualitative feedback.

The goal is to produce evidence-based design guidelines for LLM-driven interfaces that improve personalization and user experience.

Problem Statement

Learning dashboards have become popular but fall short due to their static natureâ€”presenting data in the same format regardless of user needs.

This generic approach fails to support meaningful learning, especially for users with diverse goals. LLMs offer new opportunities for dynamic UI components that adapt content and layout based on individual user behavior.

Research Questions

Main Research Question:

How does the performance and user experience of LLM-generated UI modules compare to traditionally designed static modules in learning dashboards?

How do users perceive the usability, personalization, and trustworthiness of LLM-generated modules compared to static ones?

  • User confidence in AI-generated interfaces
  • Perceived reliability and consistency
  • Learning curve and adaptation time
  • Error handling and user feedback

Do users interact more frequently or effectively with AI-generated modules?

  • Task completion times and accuracy
  • Interaction logs (clicks, time on task)
  • System Usability Scale (SUS) scores
  • Engagement frequency and patterns

What practical challenges arise when designing, deploying, and evaluating LLM-based modules in an existing platform like Strive AI?

  • Technical integration challenges
  • Design and deployment considerations
  • Evaluation methodology in real-world settings
  • Scalability and maintainability

Methodology

This study will follow a mixed-methods approach, combining a comparative user experiment with interviews and survey data.

Phase 1: Module Design & Implementation

In partnership with Digital Ops, I will help create two dashboard modules:

Module A (Static)

A fixed UI built using traditional best practicesâ€”same layout and content for all users.

Module B (LLM-Generated)

A dynamic UI that uses LLMs to personalize summaries, highlight relevant data points, and even adapt visuals depending on user data.

Phase 2: User Study

Around 16â€“20 participants will take part in an A/B test:

  • Group A: Will use the static module
  • Group B: Will use the LLM-based module
  • Tasks will be standardized (e.g., identifying personal progress, setting new goals)
  • Both groups will complete a set of interactions on the Strive AI staging dashboard

Phase 3: Data Collection & Analysis

Quantitative Data:

  • System Usability Scale (SUS)
  • Task completion times
  • Interaction logs (clicks, time on task, engagement frequency)

Qualitative Data:

  • Post-task questionnaires
  • Semi-structured interviews to gather open-ended feedback
